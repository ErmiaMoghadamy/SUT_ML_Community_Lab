## 1. Introduction: The Machine Learning Landscape
Presented by: Ermia Moghadami

- Definition of Machine Learning: An exploration into how computers can learn from data without being explicitly programmed. The discussion moved from traditional rule-based programming to data-driven approaches.
- Motivation: Analysis of why Machine Learning is essential for complex problems (like spam filtering) where manual rules are too numerous or complex to maintain.
- Applications: Discussion on real-world use cases and how ML provides solutions in fluctuating environments.

---

## 2. Part I: Types of Machine Learning Systems

### Supervised vs. Unsupervised Learning
Presented by: Ermia Moghadami
- Supervised Learning: Focused on systems where the model is trained on labeled data (e.g., Classification and Regression).
- Unsupervised Learning: Discussed how systems try to learn without a teacher, focusing on Clustering, Visualization, and Dimensionality Reduction.
- Key Differences: A comparative analysis was provided regarding data labeling and the primary objectives of each system.

### Batch and Online Learning
Presented by: Taha Mohammadi
- Batch Learning (Offline): Explored systems that must be trained using all available data, which can be computationally expensive and time-consuming.
- Online Learning (Incremental): Discussed the ability of a system to learn incrementally by feeding it data instances individually or in small groups (mini-batches), making it ideal for systems receiving continuous data streams.

### Instance-Based vs. Model-Based Learning
Presented by: Taha Mohammadi
- Instance-Based: Learning by heart; the system generalizes to new cases by comparing them to learned examples using a similarity measure.
- Model-Based: Building a regular model from a set of examples, then using that model to make predictions (e.g., Linear Regression).
- Pros and Cons: Evaluated the trade-offs between memory usage and generalization capabilities.

---

## 3. Part II: Main Challenges of Machine Learning
Presented by: Ermia Moghadami

- Data Quantity & Quality: Discussed the "Garbage In, Garbage Out" principle. Even the best algorithms cannot overcome insufficient or poor-quality data.
- Non-representative Data: The importance of ensuring training data represents the cases you want to generalize to (avoiding sampling bias).
- Irrelevant Features: The necessity of Feature Engineering—selecting and extracting the most useful features for the model.

### Overfitting and Underfitting
Presented by: Hamed Mirzaei
- Overfitting: When a model performs well on training data but fails to generalize to new data (the model is too complex).
- Underfitting: When the model is too simple to learn the underlying structure of the data.
- Solutions: Discussed regularization, simplifying models, and gathering more data to mitigate these fundamental problems.

---

## 4. Part III: Testing and Validation
Presented by: Hamed Mirzaei

- Hyperparameter Tuning:* Explored how to select the best model by fine-tuning its parameters using a validation set.
- *Data Mismatch: Addressed the challenge when the validation/test data does not perfectly match the data the model will see in production.
- Model Selection: Strategies for evaluating which algorithm performs best for a specific task.
