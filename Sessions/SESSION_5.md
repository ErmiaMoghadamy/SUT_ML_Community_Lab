## 🕒 Chronological Order of Events

### 1. Guest Presentation (15:00 – 15:40)
The session commenced with a specialized presentation by Vesal Joolanejad, focusing on the strategic decisions involved in deploying AI models.

Topic:* AI Infrastructure Design: API vs. GPU Server
*Overview: This session analyzed the trade-offs between using managed AI services (APIs) for rapid deployment versus owning and managing dedicated hardware (GPU Servers) for full control and long-term cost efficiency.
Key Analysis Points:
- Comparison of cost-efficiency and operational expenditure.
- Performance benchmarks and latency analysis.
- Scalability challenges and operational complexity.

### 2. Book Study: Section 4 Review (15:40 – 16:40)
Following the guest lecture, the community proceeded to discuss the theoretical foundations of Section 4, focusing on linear models and optimization.

| Topic | Presenter | Description |
| :--- | :--- | :--- |
| Linear Regression | Melika Arabsalmani | Discussed the Normal Equation, a closed-form analytical solution to find optimal weights without iteration, and its computational complexity. |
| Gradient Descent | Alin Kakaei | Explored the core optimization algorithm, comparing Batch, Stochastic (SGD), and Mini-batch approaches for minimizing the cost function. |
| Polynomial Regression | Ermia Moghaddami | Analyzed how to model non-linear relationships by transforming features into higher-degree polynomials. |
| Learning Curves | Erfan Jenab | Examined visual tools used to diagnose bias vs. variance and determine if a model is underfitting or overfitting. |
| Regularized Linear Models | Erfan Jenab | Covered Ridge, Lasso, and Elastic Net techniques, which add penalty terms to prevent overfitting, along with the Early Stopping strategy. |
| Logistic Regression | Taha Mohammadi | Detailed the process of estimating probabilities for binary classification and the design of Decision Boundaries. |

### 3. Practical Implementation (16:40 – 17:00)
To conclude the session, a practical demonstration was conducted to bridge the gap between mathematical theory and software engineering.
- Implementation: Logistic Regression
- Language: Rust (Chosen for its memory safety and high execution performance)
- Developer: Ermia Moghaddami
- Overview: A hands-on demonstration of how to implement the logistic regression algorithm from scratch using a systems-level language to achieve maximum efficiency.
